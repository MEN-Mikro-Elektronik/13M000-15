Attach to a VME interrupt vector.

This routine attaches the calling thread to a VME interrupt vector that occurs on the specified level. If this is the first attach on that level, it implicitely unmasks that VME level in the VME bridge.

When the interrupt occurs, the VME server masks the VME interrupt level and send the event to your attached thread. Your thread must then clear the interrupt in the interrupting VME device (if it was not implicitely cleared by the VME IACK cycle) and must then call MenC_VmeIrqLevelUnmask() to unmask the VME interrupt level again.

The event is sent from the VME server using MsgDeliverEvent(). The event types that can be sent are typically

  • SIGEV_PULSE: must be received using MsgReceivePulse() or
  • SIGEV_SIGNAL: received by a signal handler

Note that you cannot use SIGEV_INTR.

The following code snippet illustrates how to receive a SIGEV_PULSE when the interrupt occurs: 

    fd = MenC_VmeAttach();
    // create a channel on which we can receive the pulse 
    chid = ChannelCreate(0);
    coid = ConnectAttach( 0, 0, chid, _NTO_SIDE_CHANNEL, 0 );

    SIGEV_PULSE_INIT( &event, coid, SIGEV_PULSE_PRIO_INHERIT, 
                      SI_MINAVAIL, 0 );

    intId = MenC_VmeInterruptAttachEvent( fd, vector, level, &event, 0 );

    while(1){ 
        struct _pulse pulse;

        rcvid = MsgReceivePulse( chid, &pulse, sizeof(pulse), NULL );
        ...
        // handle the interrupt
        MenC_VmeIrqLevelUnmask( fd, level, intId );
    }
To detach from the vector/level, call MenC_VmeInterruptDetachEvent(). Note that the detach is automatically performed when you close the communication channel to the VME server (e.g. when your program exits).

Note that more than one client can attach to the same vector.

The client threads shall have a priority below the vme-server's interrupt thread (typically 40), otherwise strange results may occur

>=0 = interrupt ID or negative error number on error

int MenC_VmeIssueIrq (  int  fd,
int  level,
int  vector
) 
 

Trigger a VMEbus interrupt.

Triggers an interrupt on the VMEbus at the specified level. When IACK is received places the specified vector on the bus.

If interrupter busy, returns EBUSY.

This function does not wait until the vector has been fetched by another CPU. You may call MenC_VmeCheckIrq() to see which levels are still pending.

Parameters:
fd handle (value returned by MenC_VmeAttach())
level VME interrupt level (1..7)
vector VME interrupt vector (0..255)
Returns:
error number
